### 3.1 Gerarchia delle classi delle eccezioni: checked vs unchecked

In Java, tutte le eccezioni derivano dalla classe base `Throwable`, che si divide principalmente in due sottoclassi: `Exception` e `Error`. La classe `Exception` rappresenta condizioni anomale che un programma puÃ² gestire, mentre `Error` rappresenta problemi gravi che la maggior parte delle applicazioni non dovrebbe tentare di gestire, come errori del sistema o della JVM.

Le eccezioni che ereditano da `Exception` possono essere ulteriormente classificate in due categorie:

1. **Checked Exceptions (eccezioni verificate)**: Queste sono le eccezioni che il compilatore obbliga a gestire, o tramite un blocco `try-catch` o tramite la dichiarazione `throws` nel metodo. Le checked exceptions rappresentano condizioni che un programma *dovrebbe* prevedere e gestire. Un esempio classico di una checked exception Ã¨ `IOException`, che si verifica durante operazioni di input/output come la lettura o scrittura di file.
   
   ```java
   try {
       FileReader reader = new FileReader("file.txt");
   } catch (FileNotFoundException e) {
       // Gestione dell'eccezione
       System.out.println("File non trovato: " + e.getMessage());
   }
   ```

2. **Unchecked Exceptions (eccezioni non verificate)**: Le unchecked exceptions sono quelle che non richiedono un controllo obbligatorio da parte del compilatore. Sono solitamente il risultato di errori di programmazione, come tentare di accedere a un indice fuori dai limiti di un array o a un riferimento nullo. Le unchecked exceptions ereditano dalla classe `RuntimeException`. Esempi tipici includono `NullPointerException`, `ArrayIndexOutOfBoundsException` e `ArithmeticException`.
   
   ```java
   int[] numbers = {1, 2, 3};
   System.out.println(numbers[5]); // ArrayIndexOutOfBoundsException
   ```

#### Differenze principali:

- **Checked Exceptions**: Devono essere dichiarate o gestite, sono prevedibili e solitamente legate a problemi esterni al programma (ad esempio, operazioni di I/O o accesso a database).
- **Unchecked Exceptions**: Non richiedono gestione obbligatoria e sono spesso il risultato di bug nel codice che il programmatore dovrebbe risolvere direttamente.

### Schema della gerarchia delle eccezioni:

- `Throwable`
  - `Error`: Situazioni gravi che normalmente non dovrebbero essere gestite dall'applicazione.
    - `OutOfMemoryError`
    - `StackOverflowError`
  - `Exception`: Situazioni che possono essere gestite dall'applicazione.
    - `IOException` (checked)
    - `SQLException` (checked)
    - `RuntimeException` (unchecked)
      - `NullPointerException`
      - `ArrayIndexOutOfBoundsException`
      - `ArithmeticException`
